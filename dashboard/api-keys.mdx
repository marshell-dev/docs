API-ключи в chifu — это способ программно запускать сканы и забирать
результаты, не открывая дашборд. Управляются на странице **Dashboard →
Settings → API & Webhooks**.

## Первый ключ

Первый ключ выдаётся **автоматически при регистрации**. Сразу после
сигнапа на дашборде наверху появится **амбер-баннер**:

> Copy your key now — it won't be shown again.

Внутри баннера — plaintext-значение ключа в формате `chf_...`. **Это
единственный раз, когда chifu показывает ключ в открытом виде** —
сохраните его в менеджер паролей сразу. chifu хранит у себя только
bcrypt-хеш и не может восстановить ключ, если вы его потеряете.

Баннер живёт **5 минут**, потом исчезает. Если случайно закрыли — см.
ниже про выпуск нового ключа.

## Выпустить ещё ключ

1. **Dashboard → Settings → API & Webhooks**.
2. Опционально введите **Name** (например, «CI / production scans» или
   «Local dev») — это просто метка, чтобы потом было понятно, что
   откуда.
3. Нажмите **New key**.

В баннере наверху страницы покажется plaintext нового ключа — снова
один раз. Скопируйте сразу.

В таблице ключей появится новая строка:

| Колонка | Значение |
|---|---|
| **Name** | То, что вы ввели; иначе — пусто |
| **Prefix** | Первые символы ключа (`chf_k_XXX…`) для опознавания |
| **Created** | Дата выпуска |
| **Last used** | Когда ключ последний раз использовался для запроса |

## Скопировать / отозвать

- **Copy** (иконка) — копирует префикс ключа. Скопировать **полное**
  значение можно только в момент выпуска (одноразовый баннер). После
  того, как баннер исчез, в дашборде доступен только префикс — этого
  достаточно, чтобы понять, какой ключ перед вами, но не для запросов.
- **Revoke** (иконка корзины) — отзыв ключа. Сразу после отзыва все
  запросы с этим ключом начнут получать 401. Действие нельзя отменить;
  chifu спросит подтверждение.

## Безопасность

- **Не коммитьте** ключи в git, не пишите их в Slack, не передавайте по
  email.
- На стороне CI используйте **secret-переменные** провайдера (GitHub
  Secrets, GitLab CI Variables, Vercel Environment Variables и т.п.).
- При подозрении на утечку — отзовите ключ и выпустите новый.

## Что входит в права ключа

API-ключ действует **от имени вашей организации** с теми же правами,
которые есть у вашего аккаунта. Это значит:

- Запросы списываются из той же месячной квоты сканов (см.
  [Plans](/dashboard/plans)).
- Через ключ можно сканировать только домены, которые верифицированы в
  вашей организации.
- Ключ не даёт доступа к биллингу и управлению членами команды.

## FAQ

<AccordionGroup>
  <Accordion title="Можно ли восстановить ключ, если я его потерял?">
    Нет. chifu хранит только bcrypt-хеш. Выпустите новый ключ и
    отзовите старый.
  </Accordion>
  <Accordion title="Сколько ключей можно завести?">
    Лимита нет, но имеет смысл выдавать по ключу на каждую систему
    (production CI, staging CI, локальный dev) — так проще отозвать
    конкретный при компрометации.
  </Accordion>
  <Accordion title="Истекают ли ключи?">
    Не автоматически — ключ работает, пока его не отзовут вручную.
  </Accordion>
</AccordionGroup>
