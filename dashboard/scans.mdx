Страница **Dashboard → Scans** — главное рабочее место. Отсюда вы
запускаете сканы, видите их историю в реальном времени и переходите на
страницу результата.

## Запустить скан

Карточка **Run scan now** слева:

1. В dropdown'е **Domain** выберите один из ваших верифицированных
   доменов. Если доменов ещё нет — там будет ссылка на
   [Domains](/dashboard/domains).
2. Нажмите **Scan**.

Под кнопкой — бейджи **17 checks** и **~8 min**: столько инструментов
chifu может запустить и примерно столько займёт типичный скан.

<Tip>
  Сканировать можно только верифицированные домены. Это требование
  безопасности: иначе chifu стал бы публичным DDoS-инструментом.
</Tip>

## Запланировать скан

Карточка **Schedule scan** справа — то же самое, но с отложенным
запуском:

- **Domain** — какой домен сканировать.
- **Date** — календарь, выберите дату.
- **Time** — время (24-часовой формат, по умолчанию 09:00).
- **Repeat** — `Once` / `Daily` / `Weekly` / `Monthly`.

Полноценный планировщик с календарной сеткой — на отдельной странице
[Schedule](/dashboard/schedule).

## Scan history

Внизу страницы — таблица последних сканов. Колонки:

| Колонка | Что показывает |
|---|---|
| **Status** | `Queued` → `Running` → `Completed` / `Failed` / `Scheduled` |
| **Target** | Домен, который сканировался |
| **Scan ID** | Первые 8 символов идентификатора (полный — на странице скана) |
| **Started** | Когда стартовал |
| **Duration** | Сколько шёл — минуты и секунды |
| **Findings** | Сколько find'ов в итоге |

Список обновляется **в реальном времени**: статус и счётчик findings'ов
меняются прямо на ваших глазах, перезагружать страницу не нужно.

По умолчанию видны последние 3 — кнопка снизу раскрывает остальные.

Клик по строке открывает детальную страницу скана.

## Страница скана

Адрес: `/dashboard/scans/{id}`. Что вы там увидите:

### Шапка

- Домен и **статус-бейдж**.
- Timestamp + длительность скана (или «running since…», если идёт).

### Score (если скан завершён успешно)

- Слева — **Risk gauge** (полукруг 0..99, цвет по band).
- Справа — **Score breakdown**: пять строк (Vulnerabilities, TLS,
  Headers, Exposure, Posture) с прогресс-баром, значением и дельтой
  относительно предыдущего скана. Клик по строке раскрывает список
  «hits» — что именно ударило по этой подоценке.

Подробнее: [Risk score](/concepts/risk-score).

### Tech stack

Таблица обнаруженных технологий с категориями (frontend / backend /
CMS / hosting / analytics) и версиями.

### Known vulnerabilities

Таблица CVE: ID, CVSS, краткое описание. Появляется только если recon
обнаружил версии, для которых есть совпадения в CVE-базе, или если
nuclei напрямую сработал на шаблон-CVE.

### Other findings

Все остальные finding'и (SQLi, XSS, misconfig, exposure) с фильтрацией
по severity. Подробнее: [Findings](/concepts/findings).

### Tools

Сырой вывод каждого инструмента: код возврата, длительность, stdout (с
обрезанием по лимиту), либо `skipped: <reason>`. Полезно для аудита и
для понимания, почему какой-то check не дал результата.

## Статусы

| Статус | Что значит |
|---|---|
| `Queued` | Скан в очереди, ждёт свободный worker |
| `Running` | Выполняется прямо сейчас |
| `Completed` | Завершён успешно, можно открыть результат |
| `Failed` | Не доехал до конца — обычно ошибка инфраструктуры или таймаут |
| `Scheduled` | Запланирован на будущее время |

Жёсткий лимит на один скан — **15 минут**. Если упёрся в потолок,
получит `Failed`, но частичные результаты всё равно сохранятся, и их
можно открыть.
