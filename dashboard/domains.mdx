Чтобы сканировать сайт, его сначала нужно добавить в **Dashboard →
Domains** и подтвердить, что вы им владеете. chifu не будет сканировать
не подтверждённые домены — это базовое требование безопасности.

## Зачем верификация

- chifu — активный сканер: он шлёт payload'ы в формы, сканирует порты,
  делает path-fuzzing. Запускать это против чужих сайтов незаконно и
  опасно.
- Подтверждение через DNS или файл проверяет, что у вас есть управление
  доменом.
- Без верификации недоступны:
  - запуск сканов,
  - расписание,
  - firewall bypass.

## Добавить домен

1. На **Dashboard → Domains** нажмите **Add domain**.
2. Введите хост — `example.com` или `app.example.com`. Без `https://`
   и без слешей.
3. Нажмите **Add**.

Появится карточка домена со статусом **Pending** (серый бейдж с
часами).

## Подтвердить владение

Раскройте карточку — внутри панель верификации с двумя вкладками.

<Tabs>
  <Tab title="DNS TXT">
    Добавьте у регистратора DNS-записей TXT-запись:

    | Поле | Значение |
    |---|---|
    | Host / Name | `_chifu.{ваш-домен}` |
    | Type | `TXT` |
    | Value | `chifu-verify=...` (полное значение скопируйте кнопкой Copy) |

    Дождитесь пропагации (обычно от 30 секунд до нескольких минут) и
    нажмите **Verify**.

    Если регистратор требует только последний сегмент в поле host
    (например, `_chifu` вместо `_chifu.example.com`) — это нормально,
    он сам подставит зону.
  </Tab>
  <Tab title="File">
    Положите файл `chifu-token.txt` в публичную папку сайта так, чтобы
    он отдавался по `https://{ваш-домен}/chifu-token.txt`. Содержимое
    файла — только токен из дашборда (одна строка, без новой строки в
    конце).

    На Next.js / Vercel — положите файл в `/public/chifu-token.txt`.

    Нажмите **Verify**.
  </Tab>
</Tabs>

После успеха бейдж станет зелёным **Verified**.

### Если verification failed

- **DNS:** дайте больше времени на пропагацию; проверьте через
  `dig TXT _chifu.example.com`, что запись видна. Иногда регистратор
  дописывает зону дважды (`_chifu.example.com.example.com`) — исправьте
  на правильный host.
- **File:** убедитесь, что файл доступен публично без авторизации, что
  Content-Type текстовый, что в начале нет BOM, в конце — лишних
  пробелов или новой строки.

### Сменить токен

Если токен утёк (например, попал в публичный репозиторий) — нажмите
**Regenerate token**. Старый перестанет работать, а старая DNS-запись
или файл больше не подтвердят владение.

## Поддомены

Под верифицированным root-доменом сразу доступна кнопка **Add
subdomain**. Поддомены **наследуют верификацию** — вам не нужно отдельно
подтверждать `api.example.com`, если уже подтверждён `example.com`.

В UI поддомены показываются с отступом и иконкой сети под root-карточкой.
Каждый поддомен можно так же раскрыть и для каждого отдельно
настроить [firewall bypass](/dashboard/firewall-bypass), если у вас на
поддомене другая защита.

## Удалить домен

В шапке карточки домена (при наведении) появляется иконка корзины. Клик
удаляет домен и **все связанные с ним сканы, finding'и и расписания**.
Действие нельзя отменить — chifu спросит подтверждение.

## FAQ

<AccordionGroup>
  <Accordion title="Можно ли добавить IP-адрес вместо домена?">
    Нет, только полноценный hostname с DNS. IP не может быть
    подтверждён через DNS TXT.
  </Accordion>
  <Accordion title="Что если домен на бесплатном subdomain-провайдере (например, .vercel.app)?">
    Работает — используйте File verification, потому что DNS у такого
    домена вы не контролируете.
  </Accordion>
  <Accordion title="Можно ли подтвердить wildcard (`*.example.com`)?">
    Не нужно — root верификация уже распространяется на все поддомены.
    Просто добавьте root и потом добавляйте subdomains по мере необходимости.
  </Accordion>
</AccordionGroup>
