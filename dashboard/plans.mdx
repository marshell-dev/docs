chifu доступен на трёх планах. План назначается на уровне организации
и распространяется на всех её участников.

## Сравнение

| Что | **Free** | **Pro** | **Max** |
|---|---|---|---|
| Сканов в месяц | **3** | **30** | **120** |
| Сканов в день (Schedule) | — | 1 | 4 |
| [Расписания](/dashboard/schedule) | — | ✓ | ✓ |
| Ручные сканы | ✓ | ✓ | ✓ |
| [Firewall bypass](/dashboard/firewall-bypass) | ✓ | ✓ | ✓ |
| Поддомены под root | ✓ | ✓ | ✓ |
| [API-ключи](/dashboard/api-keys) | ✓ | ✓ | ✓ |
| Команда (coming soon) | — | ✓ | ✓ |

## Free

Цена: **$0**. Подходит, чтобы попробовать chifu на собственном сайте.

- **3 скана в месяц** — это значит три полных запуска pipeline'а, не
  важно, ручных или запланированных.
- **Расписания заблокированы** — на странице
  [Schedule](/dashboard/schedule) сетка покрыта оверлеем `Scheduling is
  a Pro feature` с кнопкой Upgrade.

## Pro

Цена: **$7.99/мес**. Подходит для индивидуальных разработчиков и
небольших проектов.

- **30 сканов в месяц** — примерно один скан в день, плюс запас на
  ручные ad-hoc проверки.
- **До 1 запланированного скана в день** — то есть `daily` cron работает,
  а несколько срабатываний в сутки от разных расписаний — нет.

## Max

Цена: **$23.99/мес**. Для активного мониторинга нескольких production-доменов.

- **120 сканов в месяц** — это уже 4 запланированных скана в день, или
  один скан каждые 6 часов.
- **До 4 запланированных сканов в день** — можно держать несколько
  расписаний с разной частотой и не упираться в дневной лимит.

## Как считаются сканы

Один **запуск pipeline'а** = один скан, который списывается из месячной
квоты. Не важно:

- ручной он или из расписания;
- завершился успешно, упал или таймаутнул;
- запущен через дашборд или через [API-ключ](/dashboard/api-keys).

Счётчик сбрасывается **в начале каждого календарного месяца** (UTC).

Если квота исчерпана:

- ручной скан вернёт ошибку `monthly scan quota exceeded`;
- запланированные срабатывания будут пропускаться без выполнения, но
  блоки в календаре останутся — они снова станут активными в первый
  день следующего месяца.

## Сменить план

**Dashboard → Billing** показывает текущий план, дату следующего
списания и историю invoices. Кнопка **View plans** ведёт на страницу
выбора плана.
