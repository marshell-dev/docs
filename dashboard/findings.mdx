Страница **Dashboard → Findings** агрегирует все finding'и из всех
сканов вашей организации в одну ленту. Удобно, когда нужно посмотреть
**все open critical** или найти конкретную уязвимость не помня, в каком
скане она была.

Что такое finding и какие у него поля — на странице
[Findings (concepts)](/concepts/findings).

## Шапка

- Заголовок **Findings** и счётчик типа «N vulnerabilities across your
  attack surface».
- Строка поиска — ищет по title, target, scan ID и tool'у.
- Четыре цветных severity-чипа (Critical / High / Medium / Low) с
  числом в скобках. Клик переключает фильтр; повторный клик по
  активному — сбрасывает.
- Счётчик отфильтрованных результатов справа.

## Таблица

| Колонка | Что показывает |
|---|---|
| **Severity** | Цветной бейдж — `Critical` / `High` / `Medium` / `Low` |
| **Finding** | Title + ID + (если есть) CVE/CWE ID и CVSS |
| **Target** | Домен и путь (обрезается, если длинный) |
| **Tool** | Кто нашёл — `nuclei`, `sqlmap`, `dalfox`, `nikto`, `passive`, … |
| **Status** | `Open` / `Fixed` / `Dismissed` / `Accepted` |
| **Age** | Относительное время — «2h ago», «3d ago» |

Список **обновляется в реальном времени** — новые finding'и приезжают
по мере того, как завершаются сканы. Перезагружать не нужно.

## Detail panel

Клик по строке открывает выезжающую справа панель.

### Шапка панели

- Кнопка **«Назад»**, severity-иконка, title.
- Три действия:
  - **Mark fixed** — пометить как решённую (уйдёт из дефолтного фильтра `Open`).
  - **Assign** — назначить ответственного из команды (требует Pro+ плана).
  - **Copy link** — прямая ссылка на находку, можно отправить разработчику.

### Tabs

- **Activity** — история: когда найдено, кто менял статус, к чему была
  привязана.
- **Evidence** — сырой технический вывод инструмента (запрос/ответ,
  payload). То, что нужно дать разработчику, чтобы воспроизвести.
- **Steps** — если инструмент предоставил пошаговый рецепт
  воспроизведения, он будет здесь нумерованным списком.
- **Notes** — внутренние комментарии команды по находке.

### Правая колонка (мета)

- Severity, Status, CVSS (если есть, с цветовым кодом).
- CVE ID, CWE ID (если есть).
- Tool, Target, Path.
- Scan ID, время обнаружения.
- **Description** — человеческое объяснение проблемы.
- **Recommendation** — что с этим делать (рекомендация в зелёном
  блоке).

## Статусы

| Статус | Когда использовать |
|---|---|
| `Open` | Только что найдено, в работе |
| `Fixed` | Исправлено в коде / конфиге |
| `Dismissed` | Ложное срабатывание |
| `Accepted` | Известный риск, не будем чинить |

Статус хранится глобально по finding — если та же проблема снова
найдётся в следующем скане, статус будет применён к ней автоматически
(по сигнатуре tool+type+target+title).

## Дедуп

Одна и та же уязвимость, обнаруженная в нескольких сканах подряд, не
плодит дубли в Findings — она показывается одной строкой, с временем
самого свежего обнаружения. Если уязвимость пропала из последнего скана
(например, вы её починили), она остаётся в ленте со статусом «не
найдена в последнем скане» — пока вы вручную не отметите `Fixed`.
