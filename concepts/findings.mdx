title: "Findings"

**Finding** — это конкретная проблема, обнаруженная на цели одним из
инструментов. На странице скана finding'и собраны в две таблицы
(**Known vulnerabilities** для CVE и **Other findings** для всего
остального), а на странице **Dashboard → Findings** — в одну общую
таблицу по всем сканам организации, с фильтрами.

## Поля finding'а

| Поле | Что значит |
|---|---|
| **Severity** | `critical` / `high` / `medium` / `low` / `info` |
| **Title** | Краткое описание (например, «Reflected XSS in `q` parameter» или «CVE-2023-XXXXX») |
| **Tool** | Инструмент, который её нашёл — `nuclei`, `sqlmap`, `dalfox`, `nikto`, `passive`, … |
| **Type** | Категория — см. ниже |
| **Target** | Домен и путь, где проблема воспроизводится |
| **Evidence** | Сырой кусок вывода инструмента, доказывающий находку |
| **Status** | `open` / `fixed` / `dismissed` / `accepted` — устанавливается вручную в дашборде |

## Типы

| Тип | Откуда обычно приходит |
|---|---|
| `sqli` | sqlmap |
| `xss` | dalfox, nuclei |
| `cve` | CVE-lookup (по версиям из recon), nuclei |
| `misconfig` | nuclei, nikto, passive (отсутствующие security-заголовки) |
| `cms-vuln` | joomscan, nuclei-шаблоны для CMS |
| `exposure` | ffuf (найденный `.git/config`, `.env`, `backup.zip`), passive (секреты в теле) |
| `scan-limitation` | passive (например, «Anti-bot challenge intercepted») — не уязвимость, а сигнал, что сканер видел не сам сайт, а защитный экран |

## Severity

```
critical > high > medium > low > info
```

На странице Findings под заголовком есть четыре цветных чипа
(Critical / High / Medium / Low) — клик переключает фильтр. Чтобы
сбросить, нажмите на активный чип ещё раз.

Найти конкретную проблему можно поиском в строке наверху — он ищет по
title, target, ID и tool'у.

## Детальная панель

Клик по строке открывает **detail panel** справа:

- **Activity** — история: когда найдено, кто менял статус.
- **Evidence** — сырой технический вывод инструмента, который нашёл
  проблему. Полезно, чтобы воспроизвести руками или передать разработчику.
- **Steps** — нумерованные шаги воспроизведения (если инструмент их
  даёт).
- **Notes** — место для комментариев команды.

Справа в панели — мета: severity, status, CVSS (если есть), CVE/CWE ID,
tool, target, путь, scan ID, время обнаружения, **Description** и
**Recommendation**.

## Действия

В шапке detail panel:

- **Mark fixed** — пометить как исправленную. Уйдёт из дефолтного
  списка `Open`.
- **Assign** — назначить ответственного (как только команда поднимется,
  см. [Plans](/dashboard/plans)).
- **Copy link** — прямая ссылка на эту находку.

## Что не считается finding'ом

- **Скрытые / неактивные инструменты.** Если nuclei был skipped, это не
  finding, а запись в блоке **Tools** на странице скана с
  `skipReason`.
- **Сетевая информация** (IP, ASN, страна) — это часть recon, а не
  findings.
- **Обнаруженные технологии без CVE** — попадают в **Tech stack**, но
  finding'ом становятся только когда есть совпадение с CVE.

## Связь со score

Finding'и напрямую влияют на компонент **Vulnerabilities** в [Risk
score](/concepts/risk-score) — он весит 40% от итога. Категории
`exposure` дополнительно идут в компонент **Exposure** (15%),
отсутствующие security-заголовки — в **Headers** (15%).
